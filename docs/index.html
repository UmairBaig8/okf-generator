How it works

From source to knowledge in one command

Three deterministic steps. No LLM required. Fully offline-capable.

01 🔬
Scan

tree-sitter AST parsers extract every function, class, method, and module across 13 languages — with signatures, params, docstrings, return types, and inheritance chains.

02 🔗
Link

The cross-reference linker resolves imports → dependencies and function call sites → callee concept IDs across all 17 manifest formats and supported languages.

03 ⚡
Retrieve

okf lookup WorldBankConnector returns signature, params, callers, and dependencies in milliseconds. No vector DB, no embeddings, no LLM call required.

Local AI & SLMs

Finally usable with Gemma, Llama & Phi

Cloud models have 200K token windows. Local SLMs running on a MacBook don't — they run out of memory loading an entire repo. okf lookup sends a 50-token query and gets back a 200-token concept card. No embeddings, no vector DB, no RAG pipeline.

Features

Everything you need, nothing you don't

⚡
Zero-LLM Extraction

Full AST-based extraction with tree-sitter. Deterministic, offline-capable, and resumable. No API key needed for core generation.

🔗
Cross-reference Linker

Resolves imports → dependencies and function call sites → callee IDs across all languages. Multi-hop reasoning without grep.

🌐
Interactive Visualization

Self-contained D3.js HTML dashboard. Force-directed graph, search/filter, pan/zoom, dark/light theme. Zero server, works offline.

📊
Bundle Diff

Compare two bundles and see exactly which code is affected by dependency changes. Built-in GitHub Action for PR comments.

🔌
MCP Server

Expose your knowledge bundle via the Model Context Protocol. 7 tools for lookup, caller discovery, type filtering, and bundle metadata.

🤖
Training Data Pairs

Export JSONL training pairs in 5 types (codegen, qa, doc, summarize, crosslink) compatible with any fine-tuning pipeline.
